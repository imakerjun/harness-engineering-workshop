# Harness Engineering
## AI 코드 에이전트 교육

> 삼성SDS × 멀티캠퍼스 | 1일 집합 교육 (8시간) | 2026

---

## 🎯 이 교육이 끝나면

이 워크숍의 핵심 학습 목표는 두 단계입니다.

**1단계 — 하네스 엔지니어링을 이해합니다**

"AI 에이전트의 성능을 결정하는 것은 모델이 아니라 그것을 감싸는 환경(하네스)"이라는 원리를 본인 언어로 설명할 수 있게 됩니다. 5가지 축(Context · Plan · Verify · Token · Multi-Agent)이 왜 그 5가지인지, 어떤 문제를 각각 해결하는지 짚을 수 있습니다.

**2단계 — 하네스 엔지니어링을 고려해 3가지 업무 상황에 적용할 수 있는 패턴을 연습합니다**

업무 환경은 각자 다릅니다. 워크숍 시간 안에 각자 실제 코드에 깊게 적용하는 것은 어렵습니다. 대신 **현장에서 자주 마주치는 3가지 업무 상황을 모사한 실습**으로, 워크숍 이후 본인 업무로 옮길 수 있는 손동작·프롬프트·운영 노트를 손에 익힙니다.

| 업무 상황 | 이번 워크숍에서 | 워크숍에서 손에 익히는 패턴 |
|---|---|---|
| 🔁 **같은 작업이 매번 반복될 때** | [실습 ① · 카드 B](/block3/card-b) | 매번 같은 손이 가는 작업을 Skills(슬래시 명령)로 명령어화하고, 팀이 설치로 받을 수 있는 형태(MCP)까지 한 단계 확장 설계를 잡는 패턴 |
| ➕ **기존 코드에 새 기능을 안전하게 더할 때** | [실습 ② · 카드 C](/block3/card-c) | Plan을 외부화하고, 기존 테스트를 깨지 않으면서 Verify 루프로 마무리하는 패턴 |
| 🔍 **낯선 코드를 빨리 파악해야 할 때** | [심화 자가학습 · 카드 A](/block3/card-a) | 낯선 코드(작은 코드 또는 Aider·Codex CLI 같은 큰 오픈소스) 한 모듈을 검증 가능한 사실 중심으로 파악하고, AI용 컨텍스트 파일로 정리하는 패턴. **워크숍 외 자가학습용 추가 자료** |

<Callout type="info" emoji="🎯">
**핵심 메시지**: AI 에이전트의 성능을 결정하는 것은 모델이 아니라 그것을 감싸는 환경 — **하네스(Harness)** 입니다.

`Agent = Model + Harness`

이 정의는 Birgitta Böckeler의 [Harness engineering for coding agent users](https://martinfowler.com/articles/harness-engineering.html) (Thoughtworks, 2026.02)에서 가져왔습니다.
</Callout>

## 🤔 왜 하네스인가

AI에게 전부 맡겼는데 뒷수습이 더 오래 걸린 경험, 한 번쯤 있을 것입니다.

원인은 3가지입니다:

1. **AI는 팀 컨벤션을 모릅니다** — 알려주지 않으면 추측합니다
2. **AI는 코드베이스 경계를 모릅니다** — 어떤 클래스가 실제로 있는지 모릅니다
3. **AI는 "완료"의 기준을 모릅니다** — 사람이 정의하지 않으면 스스로 허술한 기준을 만듭니다

이 3가지는 모두 **하네스가 없어서** 생기는 문제입니다.

> 우아한형제들 실제 사례: 백엔드 팀이 AI에게 10개 클래스 테스트 생성을 맡겼을 때
> **컴파일 성공률: 10%**. AI의 문제가 아니라 하네스의 부재.
>
> — [AI와 함께하는 테스트 자동화: 플러그인 개발기](https://techblog.woowahan.com/24568/), 강규한 (2025.12)

오늘 배우는 5가지 하네스 패턴:

| 축 | 핵심 질문 | 세션 |
|---|---|---|
| **Context** | AI에게 어떤 맥락을 줄 것인가 | 블록 2-1 |
| **Plan** | AI가 어떻게 실행할 것인가를 먼저 정의하는가 | 블록 2-2 |
| **Verify** | AI가 자기 결과를 확인하는 루프가 있는가 | 블록 2-3 |
| **Token** | 길어질수록 품질이 떨어지는 걸 어떻게 막는가 | 블록 2-4 |
| **Multi-Agent** | 한 에이전트에 모든 역할을 맡기고 있는가 | 블록 2-5 |

| **4. 실습 ② — 신규 기능 추가** | Codex | [카드 C](/block3/card-c) — 외부화된 plan + Verify 루프 |
