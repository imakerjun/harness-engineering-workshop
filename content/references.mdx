## 하네스 엔지니어링 (핵심 프레임워크)

### Birgitta Böckeler — Harness Engineering for Coding Agent Users

> *"The performance of an agent is determined not just by the model, but by the harness around it."*

- **제목**: [Harness engineering for coding agent users](https://martinfowler.com/articles/harness-engineering.html)
- **게재처**: Martin Fowler's bliki (Thoughtworks)
- **날짜**: 2026년 2월
- **주요 개념**: Agent = Model + Harness, 5축 하네스 프레임워크 (Context, Plan, Token, Quality, Multi-agent)
- **교육 내 인용**: 홈페이지, 블록 2 개요

## Boris Cherny — Claude Code 워크플로우

### How Boris Uses Claude Code

> *"A good plan handed to Claude is basically free — a bad plan wastes hours."*

> *"Give Claude a way to verify its work. If Claude has that feedback loop, it will 2-3x the quality of the final result."*

- **제목**: [How Boris Uses Claude Code](https://howborisusesclaudecode.com)
- **저자**: Boris Cherny (Claude Code 창시자, Anthropic)
- **날짜**: 2026
- **주요 개념**: Plan Mode 활용, Verification Loop, CLAUDE.md 컴파운딩 패턴
- **교육 내 인용**: 블록 2-1 (Context), 블록 2-2 (Plan), 블록 2-3 (Verify), 블록 4 마무리

- **교육 내 인용**: 블록 2 (에이전트 동작 원리 배경 이해)

> *"Context engineering is the art of providing the right information, in the right format, at the right time, so the model can do useful work."*

- **저자**: Andrej Karpathy (전 Tesla AI 총괄, OpenAI 공동창업자)

## 우아한형제들 — 실무 적용 사례

### 강규한 — AI 코드 에이전트 도입기 (1편: 도입과 실패)

> 백엔드 팀이 AI에게 10개 클래스 테스트 생성을 맡겼을 때 **컴파일 성공률: 10%**.
> AI의 문제가 아니라 하네스의 부재.

- **제목**: [AI와 함께하는 테스트 자동화: 플러그인 개발기](https://techblog.woowahan.com/24568/)
- **저자**: 강규한 (우아한형제들)
- **게재처**: 우아한형제들 기술 블로그
- **날짜**: 2025년 12월
- **교육 내 인용**: 홈페이지, 블록 2 개요 (하네스 필요성 사례)

### 강규한 — AI 코드 에이전트 도입기 (2편: 역할 분담으로 해결)

> PSI 플러그인(사람이 만든 템플릿 도구) + Amazon Q(AI) 역할 분담 결과:
> **생성 시간 10분 → 3분, 컴파일 성공률 10% → 95%**

- **저자**: 강규한 (우아한형제들)
- **출처**: 삼성SDS AI 코드 에이전트 특강 발표 자료 (2026.03) — [1편 블로그](https://techblog.woowahan.com/24568/)의 후속 사례
- **주요 개념**: 사람(도구)과 AI의 역할 분담 = Plan-based Execution의 실무 적용. 사람이 구조를 잡고 AI가 내용을 채우는 패턴
- **교육 내 인용**: 블록 2-2 (Plan-based Execution)

### 조현석 — MCP stdio로 AI 자산 공유

> *"공유가 아니라 설치로 전파"*

- **제목**: [흩어져 있는 AI 자산, 'MCP stdio'로 헤쳐모여!](https://techblog.woowahan.com/25986/)
- **저자**: 조현석 (우아한형제들)
- **게재처**: 우아한형제들 기술 블로그
- **날짜**: 2026년 3월
- **주요 개념**: MCP stdio를 통한 팀 AI 자산(프롬프트, 규칙) npm 패키지화. CLAUDE.md 팀 공유의 실무 확장 형태
- **교육 내 인용**: 블록 3 카드 B (반복 작업 자동화)

### 김사랑 — AI Dive Deep: 조직 단위 AI 전환

> *"업무 지시서를 써와라"* — AI에게 일을 맡기기 전, 사람이 먼저 요구사항을 구체화하는 훈련.
> 5주 만에 팀 전체의 AI 활용 방식이 달라졌다.

- **저자**: 김사랑 (우아한형제들)
- **출처**: 삼성SDS AI 코드 에이전트 특강 발표 자료 (2026.03) — 우아한형제들 내부 AI Dive Deep 세션 사례
- **주요 개념**: Context Engineering을 조직 단위로 적용. "업무 지시서"는 CLAUDE.md의 조직적 버전. 또한 1,300개 파일 이름 변경 작업을 AI로 3분 내 완료한 실무 사례 포함
- **교육 내 인용**: 블록 2-1 (Context), 블록 4 (팀 적용 로드맵)

### 배민 디자인팀 — 에센스 기반 일관된 변형 생성

> 스컬피(Sculpy) 점토 원형을 기준으로 모든 변형을 생성하는 조각 방식에서 착안.
> 하나의 기준 에센스를 정의하면 AI가 일관된 변형을 자동 생성한다.

- **출처**: 삼성SDS AI 코드 에이전트 특강 발표 자료 (2026.03) — 우아한형제들(배민) 디자인팀 사례
- **주요 개념**: Multi-agent 패턴의 실무 적용. 에센스(핵심 기준)를 사람이 정의하고, AI가 변형을 생성하는 역할 분담. 반복 작업 자동화의 고급 형태
- **교육 내 인용**: 블록 3 카드 B 참고 사례

## 성인 학습 이론

### Malcolm Knowles — Andragogy (성인 학습 원리)

> 성인은 **자신의 필요를 인식할 때** 학습 동기가 생긴다. 원리보다 문제가 먼저여야 한다.

- **저자**: Malcolm Knowles
- **출처**: *The Modern Practice of Adult Education: Andragogy versus Pedagogy* (1970)
- **주요 개념**: 자기주도 학습, 경험 기반 학습, 문제 중심 접근
- **교육 내 인용**: 블록 1 (불편함 뽑기 — 왜 문제를 먼저 발견하는가)
