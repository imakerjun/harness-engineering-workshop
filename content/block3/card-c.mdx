# 카드 C: 새 기능을 안전하게 추가하기 ★★☆

> **대상**: 기능 구현이 주 업무이고 AI를 더 안전하게 쓰고 싶은 개발자

## 시나리오

> 기존 코드에 새 기능을 추가해야 한다.
> Plan Mode + Verification Loop로 기존 동작을 깨지 않고 안전하게 추가한다.

<Callout type="info">
**어떤 하네스 패턴을 쓰는가**

- **Context**: CLAUDE.md로 기존 컨벤션 유지
- **Plan**: Plan Mode로 범위와 순서 먼저 확정
- **Verify**: 기존 테스트 + 새 테스트 모두 통과 확인
</Callout>

---

## 요구사항 카드

아래 중 하나를 선택하거나, 블록 1에서 뽑은 본인의 문제를 사용하세요.

**옵션 1 — 페이지네이션 추가**
```
GET /projects 엔드포인트에 페이지네이션을 추가하라.
- query parameter: page (기본값 1), size (기본값 10)
- 응답에 total, page, size, items 포함
- page나 size가 0 이하이면 400 반환
```

**옵션 2 — 상태 전이 검증**
```
프로젝트 상태 변경에 규칙을 추가하라.
- todo → in_progress → done 순서만 허용
- archived는 any → archived 가능
- 규칙 위반 시 400 반환 + 현재 상태와 시도한 상태 메시지 포함
```

**옵션 3 — 검색 기능 추가**
```
GET /projects에 검색 기능을 추가하라.
- query parameter: q (이름 또는 설명에 포함된 문자열)
- query parameter: status (해당 상태만 필터)
- 두 조건 동시 사용 가능
```

## Phase 1: 요구사항 분석 (30분)

<Steps>
### 직접 먼저 분석

구현 전에 직접 생각해보세요:
- 이 기능에서 예상되는 엣지 케이스는?
- 건드리면 안 되는 기존 동작은?
- 완료 기준은 무엇인가?

### AI에게 분석 요청

```
아래 요구사항을 구현하기 전에,
예상되는 엣지 케이스와 기존 코드에서 영향받을 부분을 분석해줘.

{선택한 요구사항 붙여넣기}
```

비교: 내가 생각한 것 vs AI가 발견한 것
</Steps>

---

## Phase 2: Plan Mode로 설계 (45분)

<Steps>
### Plan Mode 진입

`Shift + Tab` → Plan Mode

```
다음 요구사항을 구현하기 위한 계획을 만들어줘:
{선택한 요구사항}
```

### 계획 검토

계획이 나오면 실행 전에 반드시 확인:

- [ ] 변경할 파일 목록이 명시됐나?
- [ ] 기존 테스트를 깨지 않는다고 명시했나?
- [ ] 완료 확인 방법(테스트 실행)이 포함됐나?
- [ ] 내가 분석한 엣지 케이스가 계획에 반영됐나?

빠진 것이 있으면 수정 요청:

```
계획에 엣지 케이스 처리({엣지 케이스 내용})를 추가해줘.
그리고 완료 후 기존 테스트가 모두 통과하는지 확인하는 단계도 포함해줘.
```

### 승인 후 실행
</Steps>

## Phase 3: 구현 + 검증 (60분)

<Steps>
### 실행 후 검증 요청

```
구현이 끝나면 pytest를 실행하고,
모든 테스트가 통과할 때까지 수정해줘.
기존 테스트가 깨지면 반드시 알려줘.
새 기능에 대한 테스트도 함께 작성해줘.
```

### 엣지 케이스 테스트

```
{선택한 기능}에서 발생할 수 있는 경계 케이스를
직접 테스트하고 결과를 알려줘.

예: page=0, size=-1, 존재하지 않는 상태값 등
```

### CLAUDE.md 업데이트

이번 구현에서 발견한 것을 기록:

```markdown
## 자주 하는 실수
- 페이지네이션 추가 시 기존 목록 API 테스트가 깨짐
  → 테스트에서 page/size 기본값 처리 필요
```
</Steps>
