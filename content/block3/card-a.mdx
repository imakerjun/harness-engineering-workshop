# 카드 A: 낯선 코드 빠르게 파악하기 ★☆☆

> **대상**: 코드 탐색·문서화·테스트 추가에 어려움을 느끼는 개발자

## 시나리오

> 담당자가 퇴사한 모듈을 인수받았다. 문서도 없고 테스트도 없다.
> 이 코드를 파악하고, 핵심 기능에 테스트를 추가해야 한다.

<Callout type="info">
**어떤 하네스 패턴을 쓰는가**

- **Context**: CLAUDE.md로 "이 코드를 처음 보는 나"라는 맥락 설정
- **Verify**: 테스트를 추가하면서 AI가 직접 실행하고 검증
</Callout>

---

## Phase 1: 탐색 및 문서화 (60분)

<Steps>
### 코드베이스 전체 파악 요청

```
이 프로젝트의 전체 구조를 파악하고,
핵심 비즈니스 로직이 어디에 있는지 설명해줘.
파악이 끝나면 README.md 초안을 작성해줘.
```

### 생성된 README 직접 읽기

AI가 생성한 README.md를 직접 읽고 확인:
- 실제 코드와 다른 내용이 있나?
- 누락된 중요한 정보가 있나?

틀린 부분이 있다면 CLAUDE.md에 기록:

```markdown
## 자주 하는 실수
- README 생성 시 {구체적으로 틀린 내용} → 실제로는 {올바른 내용}
```

### 코드 구조 다이어그램 요청 (선택)

```
이 프로젝트의 주요 컴포넌트 관계를
Mermaid 다이어그램으로 그려줘.
```
</Steps>

---

## Phase 2: 테스트 추가 (90분)

<Steps>
### 테스트 없는 함수 목록 파악

```
이 프로젝트에서 테스트가 없는 핵심 함수를 찾아서
우선순위 순으로 목록을 알려줘.
```

### 함수별 테스트 작성 요청

각 함수마다 아래 패턴을 반복합니다:

```
{함수명}에 대한 단위 테스트를 작성해줘.

테스트 작성 전에 이 함수가 무엇을 해야 하는지 먼저 설명해줘.
작성 후 pytest로 통과 여부를 직접 확인해줘.
실패하면 코드를 수정해서 통과시켜줘.
```

<Callout type="warning">
**주의**: AI가 None 처리, 빈 입력, 없는 ID 조회 같은 엣지 케이스를 빠뜨리는 경우가 많습니다.
테스트 목록을 보고 "이런 경우는?" 하고 추가 요청하세요.
</Callout>

### AI가 발견한 버그 기록

AI가 테스트를 작성하다가 버그를 발견하면, CLAUDE.md에 기록합니다:

```markdown
## 자주 하는 실수
- 없는 ID 조회 시 None.to_dict() 호출 → AttributeError 발생
  → 반드시 None 체크 후 404 반환
```
</Steps>

## 완료 기준

- [ ] README.md 생성 완료 (처음 보는 사람이 이해할 수 있는 수준)
- [ ] 핵심 함수 2개 이상 테스트 작성 및 통과
- [ ] CLAUDE.md에 "자주 하는 실수" 1개 이상 추가
- [ ] `pytest tests/ -v` 실행 시 모든 테스트 통과
