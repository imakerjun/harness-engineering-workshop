## Claude Code in Action

- **링크**: [Claude Code in Action](https://anthropic.skilljar.com/claude-code-in-action)
- **게재처**: Anthropic Skilljar | 2026

Anthropic이 공식 설계한 Claude Code 실습 커리큘럼. 이 워크숍의 블록 2-3 Verification Loop 세션 설계가 이 과정의 **TDD × 에이전트** 패턴을 직접 참조했다.

> 테스트를 먼저 정의하고, AI가 그 테스트를 통과하는 구현을 완성하는 패턴.
> 사람이 "무엇이 올바른가"를 결정하고, AI가 "어떻게 구현할 것인가"를 담당한다.

**하네스 연결**: Verify 축. 검증 기준을 코드(테스트)로 명시하면 AI에게 자기 결과를 확인할 방법이 생긴다.

## 더 읽을거리

[Building Effective Agents](https://www.anthropic.com/research/building-effective-agents) (Anthropic Research, 2024.12)

에이전트를 *사용하는* 관점이 아닌 *만드는* 관점의 자료이므로 이 워크숍의 직접 참조 범위 밖이다. 그러나 Orchestrator-subagents, Evaluator-optimizer 등의 패턴은 하네스의 Multi-agent 축(5축 중 마지막)과 연결되므로, 하네스 엔지니어링 전체를 이해하고 싶다면 읽어볼 만하다.
