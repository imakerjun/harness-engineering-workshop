# Böckeler — 하네스 엔지니어링 프레임워크

> *"The performance of an agent is determined not just by the model, but by the harness around it — the context it receives, the plan it follows, and the feedback loops that let it self-correct."*

**Birgitta Böckeler** | [Harness engineering for coding agent users](https://martinfowler.com/articles/harness-engineering.html) | Thoughtworks, 2026년 2월

---

## 이 자료가 이 워크숍의 토대인 이유

이 워크숍의 핵심 프레임인 `Agent = Model + Harness`가 이 논문에서 나왔다.

Böckeler는 같은 모델을 쓰더라도 결과가 달라지는 이유를 분석했다. 차이는 모델의 성능이 아니라, 모델 주변에 구축한 환경 — 즉 **하네스** — 에서 온다는 것이 핵심 주장이다.

논문은 하네스를 5축으로 분류한다:

| 축 | 내용 |
|---|---|
| **Context** | AI가 받는 맥락의 품질 |
| **Plan** | AI가 따르는 계획의 명확성 |
| **Token** | 컨텍스트 윈도우 활용 효율 |
| **Quality** | 결과 검증과 피드백 루프 |
| **Multi-agent** | 에이전트 간 역할 분담 |

이 워크숍은 즉시 효과가 큰 첫 3축(Context, Plan, Verify)에 집중한다.
