      ```bash
      claude --version
      ```

## 2. 실습 코드베이스 준비

```bash
git clone https://github.com/imakerjun/sds-harness-lab
cd sds-harness-lab
pip install -r requirements.txt
```

서버 실행 확인:

```bash
uvicorn app.main:app --reload
# → http://localhost:8000/health 접속 후 {"status":"ok"} 확인
```

테스트 실행 확인:

```bash
python3 -m pytest tests/ -v
# → "0 tests ran" 이 정상 (테스트가 아직 없음)
```

<Callout type="info">
**이 코드베이스는 의도적으로 불완전합니다.** 테스트 없음, 문서 없음, 입력값 검증 없음 — 이것이 오늘 실습의 소재입니다.
</Callout>

## 코드베이스 구조 미리보기

```
sds-harness-lab/
├── CLAUDE.md              ← 오늘 우리가 채울 파일
├── requirements.txt
├── app/
│   ├── main.py            ← FastAPI 앱 진입점
│   ├── models.py          ← 데이터 모델 (Project, Task)
│   ├── database.py        ← 인메모리 저장소
│   ├── routers/
│   │   ├── projects.py    ← 프로젝트 API 엔드포인트
│   │   └── tasks.py       ← 할일 API 엔드포인트
│   └── services/
│       ├── project_service.py
│       └── task_service.py
└── tests/                 ← 지금은 비어있음
```

**의도적 결함 목록** (실습에서 발견하고 수정할 것):

| 위치 | 결함 | 실습 세션 |
|---|---|---|
| `services/project_service.py` | 없는 ID 조회 시 500 반환 | 블록 2-3 |
| `routers/projects.py` | 입력값 검증 없음 | 블록 2-2 |
| `database.py` | 완료된 프로젝트에 할일 추가 가능 | 블록 3 카드 C |
| `services/` 전체 | 중복 로직 | 블록 3 카드 A |
| `tests/` | 테스트 없음 | 블록 2-3, 블록 3 |
